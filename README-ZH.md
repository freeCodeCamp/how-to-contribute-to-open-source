## 一般开源贡献
- [The Definitive Guide to Contributing to Open Source](https://www.freecodecamp.org/news/the-definitive-guide-to-contributing-to-open-source-900d5f9f2282/) by [@DoomHammerNG](https://twitter.com/DoomHammerNG)
- [An Intro to Open Source](https://www.digitalocean.com/community/tutorial_series/an-introduction-to-open-source) - DigitalOcean提供的指导你通过你的方式在GitHub上成功进行贡献的指南。
- [Code Triage](https://www.codetriage.com/) - 另一个，确实不错的，通过语言过滤来查找流行的开源库和issue的工具。
- [Forge Your Future with Open Source](https://fossforge.com) ($) - 書籍專注於開源項目。教你怎樣找到開源項目和開始貢獻。囊括所有軟件開發的角色，不排除只有軟件工程師。
- [Awesome-for-beginners](https://github.com/MunGell/awesome-for-beginners) - 一個GitHub開源庫，收集一些對新的貢獻來說有不錯的Bug進行貢獻的工程，並且通過標籤來進行描述。
- [Open Source Guides](https://opensource.guide/) - 為希望學習怎樣對開源項目進行貢獻的個人，組織和公司收集的資源。
- [45 Github Issues Dos and Don’ts](https://hackernoon.com/45-github-issues-dos-and-donts-dfec9ab4b612) - 在GitHub上什麼該做什麼不該做。
- [GitHub Guides](https://guides.github.com/) - 關於如何有效率的使用GitHub的基本指導。
- [Contribute to Open Source](https://github.com/danthareja/contribute-to-open-source) - 通過為一個模擬的工程貢獻來學習GitHub工作流。
- [Linux Foundation's Open Source Guides for the Enterprise](https://www.linuxfoundation.org/resources/open-source-guides/) - Linux基金會改於開源項目的指南。
- [CSS Tricks An Open Source Etiquette Guidebook](https://css-tricks.com/open-source-etiquette-guidebook/) - 一個開源規則指南, 由 Kent C. Dodds 和 Sarah Drasner 撰寫.
- [A to Z Resources for Students](https://github.com/dipakkr/A-to-Z-Resources-for-Students) - 為大學生提供的關於學習一門新語言的資源或機會的對照表。
- [Pull Request Roulette](http://www.pullrequestroulette.com/) - 這個網站有一個託管在GitHub上提交上去為了複查的屬於開源的項目的拉取請求列表。
- ["How to Contribute to an Open Source Project on GitHub" by Egghead.io](https://egghead.io/courses/how-to-contribute-to-an-open-source-project-on-github) - 一個指導你如何一步一步開始在GitHub上為開源工程做貢獻的視頻教程。
- [Contributing to Open Source: A Live Walkthrough from Beginning to End](https://medium.com/@kevinjin/contributing-to-open-source-walkthrough-part-0-b3dc43e6b720) - 這開源貢獻指南覆蓋所有課題包括選擇適合的項目，處理Issue， 和PR合併。
- ["How to Contribute to Open Source Project by" Sarah Drasner](https://css-tricks.com/how-to-contribute-to-an-open-source-project/) - 細節焦點如何用pull request (PR) 貢獻別人的項目。
- ["How to get started with Open Source by" Sayan Chowdhury](https://www.hackerearth.com:443/getstarted-opensource/) - 這文章包含如何貢獻開源項目基於新手最常用的語言。
- ["Browse good first issues to start contributing to open source"](https://github.blog/2020-01-22-browse-good-first-issues-to-start-contributing-to-open-source/) - Github幫你尋找適合你的開源貢獻中的第一次Issue
- ["How to Contribute to Open Source Project" by Maryna Z](https://rubygarage.org/blog/how-contribute-to-open-source-projects) - 這綜合文章定向於商業（對於個人貢獻者有助），闡明一些何以，如何，和為什麼要開源貢獻。
- ["start-here-guidelines" by Andrei](https://github.com/zero-to-mastery/start-here-guidelines) -
關於開源的課程，從遊樂園開始。針對教育和實制操作。
- ["Getting Started with Open Source" by NumFocus](https://github.com/numfocus/getting-started-with-open-source) - a GitHub repo that helps contributors overcome barriers to entry in open-source.
- ["Opensoure-4-everyone" by Chryz-hub ](https://github.com/chryz-hub/opensource-4-everyone) -一個跟開源有關的repository。這項目幫助Github成員透明度，基本和進階Git Commands，如何開始開源貢獻等等。
- ["Open Advice"](http://open-advice.org/) - 廣泛免費軟件計劃的知識收集，提供42個傑出貢獻者想一開始就知道的知識，從而幫助你無論何時何地也可以出力貢獻。
- ["GitHub Learning Lab"](https://lab.github.com/) - 用Github學習實驗室升級你的Github技術吧！我們友善的機械人會帶領你一系列有趣和實制的項目，讓你可以增進你的技術和分享有助的回饋。
- ["Ten simple rules for helping newcomers become contributors to open projects"](https://doi.org/10.1371/journal.pcbi.1007296) - 這文章包含從其他社團，成員經驗，領導人和觀察者收集回來的開源貢獻者規則。
- ["Step-by-Step guide to contributing on GitHub"](https://www.dataschool.io/how-to-contribute-on-github/) - 一個用視覺和結合一步一步帶領你整個開源貢獻的步驟

## Mozilla的貢獻者生態系統
- [Good First Bugs](https://bugzil.la/sw:%22[good%20first%20bug]%22&limit=0) - 開發者確認是可以比較好熟悉工程的bug。
- [MDN Web Docs](https://developer.mozilla.org/en-US/docs/MDN/Contribute) - 修理內容問題和Bugs，從而幫助MDN網路文件團隊記錄網路平台。
- [Mentored Bugs](https://bugzilla.mozilla.org/buglist.cgi?quicksearch=mentor%3A%40) - 當你在修復的時候遇到問題會有指定的人通過網上交流的方式為你的提供幫助的bug。
- [Bugs Ahoy](http://www.joshmatthews.net/bugsahoy/) - 專門發現在Bugzilla上bug的網站。
- [Firefox DevTools](http://firefox-dev.tools/) - 一個針對火狐瀏覽器開發者工具相關的bug網站。
- [What Can I Do For Mozilla](http://whatcanidoformozilla.org/) - 根據的你的技能和興趣回答你可以為Mozilla開源做一些什麼。
- [Start Mozilla](https://twitter.com/StartMozilla) - 一個為Mozilla新進貢獻者推送一些合適的Issue的Twitter賬號。

## 開源書籍
- [Producing Open Source Software](http://producingoss.com/) - Producing Open Source Software是一本關於開源開發人性化方面的書。它描述了成功的項目如何運作，用戶和開發人員的期望，以及自由軟件的文化。
- [Open Source Book Series](https://opensource.com/resources/ebooks) - 通過 https://opensource.com 提供的免費電子書的全面列表，進一步了解開源和不斷增長的開源運動。
- [Software Release Practice HOWTO](http://tldp.org/HOWTO/Software-Release-Practice-HOWTO/) - HOWTO介紹了Linux和其他開源項目的良好發布實踐。通過遵循這些實踐，您將使用戶盡可能容易地構建和使用您的代碼，並使其他開發人員能夠理解您的代碼並與您合作改進它。
- [Open Sources 2.0 : The Continuing Evolution](https://archive.org/details/opensources2.000diborich) (2005) -《開放源代碼2.0》是當今技術領導者富有洞察力和啟發性的論文集，這些論文繼續描繪s了1999年出版的《開放源代碼：革命之聲》一書中的進化圖景。
- [The Architecture of Open Source Applications](https://www.aosabook.org/en/git.html) - 現示各種Git項目底下的作業，從而賦予分佈式工作流程，Git和其他版本控制的分別
- [Open Sources: Voices from the Open Source Revolution](https://www.oreilly.com/openbook/opensources/book/) - Essays from open-source pioneers such as Linus Torvalds (Linux), Larry Wall (Perl), and Richard Stallman (GNU).

## 開源貢獻初步
- [Up For Grabs](http://up-for-grabs.net/#/) - 包含對初學者友好的項目Issue
- [First Timers Only](http://www.firsttimersonly.com/) - 標記為"first-timers-only"的Bug列表。
- [First Contributions](https://firstcontributions.github.io/) - 在5分鐘內完成您的第一次開源貢獻。幫助初學者開始貢獻的工具和教程。
- [Hacktoberfest](https://hacktoberfest.digitalocean.com/) - 鼓勵開源貢獻的計劃。通過在10月份提出至少5個請求來獲得諸如T恤和貼紙之類的禮物。
- [24 Pull Requests](https://24pullrequests.com) - 24 Pull Requests是一個在12月份促進開源協作的項目。
- [Ovio](https://ovio.org) - 一個篩選了一些貢獻者友好項目的平台。擁有 [強大的Issue尋找工具](https://ovio.org/issues)， 讓貢獻者儲存項目和Issue。
- [Google Summer of Code](https://summerofcode.withgoogle.com/) - Google資助，一年一度的項目，讓多些學生開發者一起參與開源軟件發展
- [Rails Girls Summer of Code](https://railsgirlssummerofcode.org/) - 全球女性和非二元性別團體的項目，開源者可參與現有的開源項目，從而增加技術。
- [Major League Hacking Fellowship](https://fellowship.mlh.io/) - 一個遙遠實習團體讓有志的技術家可以發展，或貢獻開源項目。
