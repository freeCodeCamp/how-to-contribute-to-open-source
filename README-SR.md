# Добро дошли нови Open Source сарадници

[![Pull Requests Welcome](https://img.shields.io/badge/PRs-welcome-brightgreen.svg?style=flat)](http://makeapullrequest.com)
[![first-timers-only Friendly](https://img.shields.io/badge/first--timers--only-friendly-blue.svg)](https://www.firsttimersonly.com/)
[![Build Status](https://api.travis-ci.org/freeCodeCamp/how-to-contribute-to-open-source.svg?branch=master)](https://travis-ci.org/freeCodeCamp/how-to-contribute-to-open-source)

Ово је листа материјала за људе који тек улазе у свет доприношења Open Source пројектима.

Ако будете пронашли неки додатни материјал на ову тему, молимо Вас додајте га и допринесите овом пројету (pull request).

Ако имате било какво питање или коментар, молимо Вас креирајте питање (issue).

## Доприношење Open Source-u - генерално
- [Целосадржани водич за доприношење Open Source-u](https://www.freecodecamp.org/news/the-definitive-guide-to-contributing-to-open-source-900d5f9f2282/) написао [@DoomHammerNG](https://twitter.com/DoomHammerNG)

- [Issuehub.pro](http://issuehub.pro/) - алатка за претрагу GitHub питања по ознакама и програмском језику.
- [Code Triage](https://www.codetriage.com/) - још једна, веома згодана, алатка за претрагу популарних репозиторијума и питања филтрираних по програмском језику.
- [Awesome-for-beginners](https://github.com/MunGell/awesome-for-beginners) - GitHub репо који прикупља грешке у апликацијама погодне за почетнике у Оpen Source доприношењу, и поставља ознаке да их детаљније опише.
- [Open Source Водичи](https://opensource.guide/) - Скуп материјала за особе, заједнице и предузећа која желе да науче како да покрену и допринесу Open Source пројекту.
- [45 савета шта радити а шта не на GitHub-u](https://hackernoon.com/45-github-issues-dos-and-donts-dfec9ab4b612) - Шта радити а шта не на GitHub-u.
- [GitHub Савети](https://guides.github.com/) - основни савети како користити GitHub ефикасно.
- [Допринети Open Source](https://github.com/danthareja/contribute-to-open-source) - Научите ток GitHub-а тако што ћете допринети програмски код симулацији пројекта.
- [Open Source Савети Linux Фондације за Предузећа](https://www.linuxfoundation.org/resources/open-source-guides/) - Open Source Савети Linux Фондације за Предузећа.
- [CSS Трикови Open Source Бонтон](https://css-tricks.com/open-source-etiquette-guidebook/) - Open Source Бонтон, написали Kent C. Dodds и Sarah Drasner.
- [A до Ш Материјали за Студенте](https://github.com/dipakkr/A-to-Z-Resources-for-Students) - Листа материјала и могућности за студенте да науче нови програмски језик.
- [Pull Request Рулет](http://www.pullrequestroulette.com/) - Овај сајт садржи листу Pull Request-ova, који припадају Open Source пројектима постављених на GitHub-u, који чекају да буду прегледани.
- ["Како допринети Open Source пројекту на GitHub" написао Egghead.io](https://egghead.io/courses/how-to-contribute-to-an-open-source-project-on-github) - Корак по корак видео водич, како на као почетник допринети Open Source пројектима на GitHub-u.
- [Допринети Open Source-u: Водич уживо од почетка до краја](https://medium.com/@kevinjin/contributing-to-open-source-walkthrough-part-0-b3dc43e6b720) - Овај водич кроз прилог како допринети Open Source-u покрива све, од одабира одговарајућег пројекта, рада на неком питању, до спајања PR-а.
- ["Како допринети Open Source пројекту" написао Sarah Drasner](https://css-tricks.com/how-to-contribute-to-an-open-source-project/) - Они се фокусирају на ситнице везане за Pull Request на нечијем пројекту на GitHub-u.
- ["Како почети са Open Source" написао Sayan Chowdhury](https://www.hackerearth.com:443/getstarted-opensource/) - Овај чланак покрива материјале за почетнике како допринети Open Source-u на основу њиховог омиљеног програмског језика.
- ["Пронађите добро прво питање да бисте почели да доприносите open source-u"](https://github.blog/2020-01-22-browse-good-first-issues-to-start-contributing-to-open-source/) - GitHub Вам сада помаже да пронађете добро прво питање да бисте почели да доприносите open source-u.
- ["Како допринети Open Source пројекту" написала Maryna Z](https://rubygarage.org/blog/how-contribute-to-open-source-projects) - Овај свеобухватни чланак је усмерена ка предузећа (али још увек је користан за поједине сараднике) у којој се говори о томе зашто, како, и којим Open Source пројектима допринети.
- ["start-here-guidelines" написао Andrei](https://github.com/zero-to-mastery/start-here-guidelines) -
Закорачимо у Git свет Оpen Source-a, тако што ћемо почети од Open Source игралишта. Посебно дизајнирано за образовање и практично искуство.
- ["Увод у Open Source" написао NumFocus](https://github.com/numfocus/getting-started-with-open-source) - GitHub repo који помаже људима који доприносе да преброде изазове након што закораче у open-source.
- ["Opensoure-4-everyone" (Opensource-за-све) написао Chryz-hub ](https://github.com/chryz-hub/opensource-4-everyone) - Repo у коме се налази све везано за open source. Ово је пројекат намењен да Вам помогне са видљивошћу међу свим члановима GitHub-a, увежбавању основних и напредних git команди, почетку рада са Open Source и још много тога.
- ["Open Advice"](http://open-advice.org/) - Скуп знања прикупљеног из широког спектра Open Source пројеката. Одговара на питање шта су 42 истакнута сарадника желела да знају када су започели, тако да бисте могли имати предност без обзира на то како и где доприносите.
- ["GitHub Лабараторија за учење"](https://lab.github.com/) - Побољшајте своје вештине са GitHub Лабараторијом за учење. Наш љубазни бот ће вас спровести кроз низ забавних, практичних пројеката да бисте за трен научили вештине које су вам потребне - и успут дали корисне повратне информације.
- ["Десет једноставних правила која помажу новајлијама да постану сарадници на Open Source пројектима"](https://doi.org/10.1371/journal.pcbi.1007296) - Овај чланак покрива правила заснована на студијама многих заједница и искуствима чланова, вођа и посматрача.

## Mozilla екосистем за сараднике
- [Грешке у апликацијама за почетнике](https://bugzilla.mozilla.org/buglist.cgi?quicksearch=sw%3Agood+first+bug) - грешке које су програмери идентификовали као добар увод у пројекат. 
- [MDN Web Docs](https://developer.mozilla.org/en-US/docs/MDN/Contribute) - помозите тиму MDN Web Docs у документовању веб платформе тако што ћете решити проблеме са садржајем и програмске грешке.
- [Mentored Bugs](https://bugzilla.mozilla.org/buglist.cgi?quicksearch=mentor%3A%40) - грешке које имају додељеног ментора који ће бити ту да вам помогне када се заглавите док радите на поправци.
- [Bugs Ahoy](https://www.joshmatthews.net/bugsahoy/) - сајт посвећен проналажењу грешака на  Bugzilla.
- [Firefox DevTools](http://firefox-dev.tools/) - сајт посвећен грешкама попуњеним за развојне алате у Firefox прегледачу.
- [Шта могу урадити за Mozilla-u](https://whatcanidoformozilla.org/) - схватите на чему можете да радите одговарајући на гомилу питања о вашим вештинама и интересовањима.
- [Start Mozilla](https://twitter.com/StartMozilla) - налог на Твитеру који твитује о темама погодним за новопечене сараднике у Mozilla екосистему.

## Open Source књиге
- [Како Направити Open Source Software](https://producingoss.com/) - Како Направити Open Source Software је књига написана о љутској страни развоја Open Source. Она описује принципе како функционише успешни пројекат, очекивања од корисника и развојних програмера и култури бесплатног software.
- [Open Source Book Серијал](https://opensource.com/resources/ebooks) - Сазнајте више о Open Source и растућем покрету Open Source кода са свеобухватном листом бесплатних е -књига са линка https://opensource.com.
- [Пракса пустања Software у рад КАКО УРАДИТИ](https://tldp.org/HOWTO/Software-Release-Practice-HOWTO/) - Овај КАКО УРАДИТИ описује добре праксе пуштања у рад за Linux и друге Open-Source пројекте. Пратећи ове упуте, олакшаћете корисницима да из buildaju ваш код и користе га, а другим програмерима да разумеју ваш код и сарађују са вама на његовом побољшању.
- [Open Sources 2.0 : Наставак Еволуције](https://archive.org/details/opensources2.000diborich) (2005) - Open Sources 2.0 је збирка проницљивих есеја и изазовних размишљања данашњих технолошких лидера која наставља да осликава еволуциону слику која се развила у књизи 1999., Open Sources: Гласови Револуције.
- [Архитектура Open Source Апликације](http://www.aosabook.org/en/git.html) - Приказује како различити аспекти GIT-a раде испод хаубе како би омогућили дистрибуиране токове и како се разликује од других система за контролу верзија (VCSs).
- [Open Sources: Гласови Open Source Револуције](https://www.oreilly.com/openbook/opensources/book/) - Есеји пионира open-source кода, као што су Linus Torvalds (Linux), Larry Wall (Perl), и Richard Stallman (GNU).

## Open Source иницијатива за доприношење
- [Up For Grabs](https://up-for-grabs.net/) - Садржи пројекте са проблемима прилагођеним почетницима
- [First Timers Only](https://www.firsttimersonly.com/) - Листа грешака које су означене "first-timers-only".
- [First Contributions](https://firstcontributions.github.io/) - Направите свој први допринос Open Source коду за 5 минута. Алат и водич који ће почетницима помоћи да започну са доприношењем. [Овде](https://github.com/firstcontributions/first-contributions) је изворни код GitHub-а за положај и могућност да дате допринос самом repou.

- [24 Pull Requests](https://24pullrequests.com) - 24 Pull Requests је пројекат за промоцију сарадње Open Source-а током децембра.
- [Ovio](https://ovio.org) - Платформа са организованим избором contributor-friendly projects. Има [веома моћну алатку за претрагу питања](https://ovio.org/issues) као и функционалност да можете да сачувате те пројекте и питања.
- [Google Лето програмирања](https://summerofcode.withgoogle.com/) - Годишњи плаћени програм направљен од стране Google-а оформљен за привлачење студената развоју open-source software.
- [Rails Girls Summer of Code](https://railsgirlssummerofcode.org/) - Глобални програм стипендирања за жена и не програмера, који раде на постојећим open-source пројектима и проширују своје вештине.
- [Светска лига друства хакера](https://fellowship.mlh.io/) - Стажирање на даљину за задивљујуће технологије, где програмирају или доприносе open-source пројектима.
